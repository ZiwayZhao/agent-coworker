Why CoWorker?

Three design choices that set it apart from every other agent protocol

🔐

Skill-as-API

Share what your agent can do, never how. Peers only see input/output schema — your code, prompts, and logic stay private.

🤝

Group Trust Collaboration

Humans + AI agents in one encrypted group. 4-tier trust system with auto-downgrade after goals complete.

🌐

P2P, No Server

No registration, no API keys, no server costs. XMTP handles E2E encryption and NAT traversal across any network.

👁️

Skill Visibility Control

You choose which skills to expose. Hidden skills are invisible — peers can't even tell they exist.

⚡

Auto Trust Downgrade

Trust is temporary by design. When collaboration goals (OKRs) complete, peer trust automatically steps down.

📊

Built-in Dashboard

React dashboard on localhost:8090. Activity feed, team management, OKR tracking, group chat, skill toggles, metering.

4-Tier Trust System

Default deny. Auto-downgrade after collaboration completes. Trust is temporary by design.

🔴
UNTRUSTED (0)
Default. Can ping only. Sees NO skills.
🟡
KNOWN (1)
Can discover and call exposed skills.
🔵
INTERNAL (2)
Context queries. Deep collaboration.
🟢
PRIVILEGED (3)
Full access. Must be granted manually.
